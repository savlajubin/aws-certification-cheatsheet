Types of Machine Learning

📋 Supervised Learning
Learns from labeled data (input + correct answer).

Two types:
• Classification — predict a category (spam/not spam, cat/dog)
• Regression — predict a number (house price, temperature)

Algorithms: Linear Regression, Logistic Regression, Decision Trees, Random Forest, SVM, XGBoost, Neural Networks
🔍 Unsupervised Learning
Learns from unlabeled data (no correct answers).

Types:
• Clustering — group similar items (customer segments)
• Dimensionality Reduction — simplify data (PCA)
• Anomaly Detection — find outliers (fraud)

Algorithms: K-Means, DBSCAN, PCA, t-SNE
🎮 Reinforcement Learning
Agent learns by trial and error — gets rewards/penalties.

• No labeled data — learns from environment
• Agent takes actions → gets reward/penalty
• Goal: maximize cumulative reward

Examples: Game playing, robotics, AWS DeepRacer, autonomous driving

Model Evaluation Metrics (IMPORTANT)

Classification Metrics

MetricFormula / MeaningWhen to Use
AccuracyCorrect predictions / Total predictionsBalanced datasets. NOT good for imbalanced data.
PrecisionTrue Positives / (True Positives + False Positives)"Of all predicted positive, how many were actually positive?" Use when false positives are costly (spam filter).
Recall (Sensitivity)True Positives / (True Positives + False Negatives)"Of all actual positives, how many did we catch?" Use when false negatives are costly (cancer detection).
F1 ScoreHarmonic mean of Precision & RecallBalance between precision and recall. Good for imbalanced datasets.
AUC-ROCArea Under the ROC CurveOverall model performance across all thresholds. 1.0 = perfect, 0.5 = random.

Regression Metrics

MetricMeaning
MSE (Mean Squared Error)Average of squared differences. Penalizes large errors more.
RMSE (Root MSE)Square root of MSE. Same units as target variable.
MAE (Mean Absolute Error)Average of absolute differences. Less sensitive to outliers.
R² (R-Squared)How much variance the model explains. 1.0 = perfect fit.

Confusion Matrix (Know This!)

Predicted PositivePredicted Negative
Actual Positive✅ True Positive (TP)❌ False Negative (FN)
Type II Error
Actual Negative❌ False Positive (FP)
Type I Error		✅ True Negative (TN)
Precision = "Don't cry wolf" (minimize false alarms). Recall = "Don't miss anything" (catch all positives). F1 = balance both.

Common ML Problems

ProblemWhat It MeansHow to Fix
OverfittingModel memorizes training data, performs poorly on new data. Too complex.More data, regularization, dropout, simpler model, cross-validation
UnderfittingModel is too simple, can't learn patterns. Poor on both training & test data.More features, more complex model, more training time, less regularization
Bias (Statistical)Systematic error from wrong assumptions. Model consistently wrong in one direction.Better features, more complex model
VarianceModel is too sensitive to training data fluctuations.More data, regularization, ensemble methods
Data LeakageTest data information leaks into training. Artificially high performance.Proper train/test split, careful feature engineering
Class ImbalanceOne class has far more samples than another (99% vs 1%).Oversampling (SMOTE), undersampling, weighted loss, F1 score
Overfitting = "too good on training, bad on test." Underfitting = "bad on everything." The exam loves asking about these.

✨ Domain 2: Fundamentals of Generative AI (24%)

What is Generative AI?

A type of AI that can create NEW content — text, images, code, audio, video — based on patterns learned from training data. Unlike traditional ML that classifies or predicts, GenAI generates.

Traditional ML

  • Classifies, predicts, detects
  • Input → Label/Number
  • "Is this a cat?" → Yes/No
  • "What's the price?" → $350K
  • Discriminative models

Generative AI

  • Creates new content
  • Input → New text/image/code
  • "Write a poem about cats" → 🐱📝
  • "Generate a house image" → 🏠🖼️
  • Generative models (LLMs, diffusion)

Foundation Models (FMs)

Large AI models pre-trained on massive datasets that can be adapted to many tasks. They are the "foundation" you build on.

🏗️ Key Characteristics
• Trained on massive, diverse datasets (internet-scale)
• General-purpose — adaptable to many tasks
• Very expensive to train from scratch ($millions)
• Can be fine-tuned or used via prompts
• Billions of parameters
📦 Foundation Models on Bedrock
• Amazon Titan — AWS's own FMs (text, image, embeddings)
• Anthropic Claude — Advanced reasoning, long context
• Meta Llama — Open-source LLM family
• Mistral AI — Efficient, high-performance
• Cohere — Text generation & embeddings
• Stability AI — Image generation (Stable Diffusion)
• AI21 Labs — Jurassic models
⚙️ How to Use FMs
1. Prompt Engineering — craft inputs to get desired outputs (cheapest, fastest)
2. RAG — augment with your own data (no retraining)
3. Fine-Tuning — retrain on your specific data (more effort)
4. Continued Pre-Training — extend training with domain data (most effort)
The exam expects you to know WHEN to use each approach: Prompt engineering first (cheapest) → RAG if you need external knowledge → Fine-tuning if you need specialized behavior.

Transformers & Attention Mechanism

The architecture behind all modern LLMs. You don't need to code it, but understand the concepts.

ConceptWhat It MeansWhy It Matters
TransformerNeural network architecture that processes all input tokens in parallelFoundation of all modern LLMs (GPT, Claude, Llama)
Self-AttentionModel weighs the importance of each word relative to every other wordUnderstands context: "bank" in "river bank" vs "bank account"
TokensPieces of text (words, subwords, characters) the model processesModels have token limits (context window). Pricing is per token.
Context WindowMaximum number of tokens a model can process at onceLarger = can handle longer documents. Claude = up to 200K tokens.
ParametersInternal weights learned during training (billions in LLMs)More parameters ≈ more capable but more expensive to run.
EncoderProcesses input and creates representationsUsed for understanding tasks (classification, embeddings).
DecoderGenerates output token by tokenUsed for generation tasks (text, code). GPT = decoder-only.
TemperatureControls randomness of output (0 = deterministic, 1 = creative)Low temp = factual/consistent. High temp = creative/varied.
Top-p / Top-kControls which tokens are considered for next predictionAnother way to control output diversity.
Temperature is a very common exam topic. Low temperature (0-0.2) = factual, consistent answers. High temperature (0.7-1.0) = creative, diverse outputs.

Prompt Engineering (HEAVILY TESTED)

The art of crafting inputs to get the best outputs from a foundation model. No retraining needed.

Prompt Techniques

TechniqueDescriptionExample
Zero-ShotAsk the model directly with no examples"Classify this review as positive or negative: 'Great product!'"
Few-ShotProvide a few examples before the actual task"Review: 'Love it' → Positive. Review: 'Terrible' → Negative. Review: 'Amazing quality' → ?"
Chain-of-ThoughtAsk the model to reason step by step"Solve this math problem. Show your reasoning step by step."
System PromptSet the model's role/behavior/constraints"You are a helpful customer service agent. Only answer questions about our products."
Retrieval-AugmentedInclude retrieved context in the prompt (RAG)"Based on the following document: [context]... Answer: ..."

Prompt Best Practices

✅ Do
• Be specific and clear
• Provide context and constraints
• Use examples (few-shot)
• Specify output format
• Break complex tasks into steps
• Set a role/persona
❌ Don't
• Be vague or ambiguous
• Assume the model knows your context
• Use overly complex prompts
• Ignore output format specification
• Skip testing different prompts
• Rely on a single prompt attempt
Zero-shot = no examples. Few-shot = a few examples. Chain-of-thought = "think step by step." These three are the most tested prompt techniques.

⚠️ Common Traps
• Bedrock ≠ SageMaker (use FMs vs build models)
• RAG ≠ Fine-Tuning (retrieve data vs retrain model)
• Precision ≠ Recall (false positives vs false negatives)
• Overfitting ≠ Underfitting (too complex vs too simple)
• Temperature 0 = deterministic, 1 = creative
• Supervised = labeled data, Unsupervised = unlabeled
• SageMaker Clarify = bias + explainability (not Guardrails)
• Bedrock data is NOT used to train base models
