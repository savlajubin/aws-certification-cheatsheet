☁️ Domain 1: Cloud Concepts (24%)

What is Cloud Computing?

On-demand delivery of IT resources over the internet with pay-as-you-go pricing. Instead of buying and maintaining physical servers, you rent them from AWS.

6 Advantages of Cloud (Memorize These!)

1. Trade CapEx for OpEx
No upfront cost. Pay only for what you use (variable expense).
2. Massive Economies of Scale
AWS buys in bulk → lower prices passed to you.
3. Stop Guessing Capacity
Scale up/down as needed. No over-provisioning.
4. Increase Speed & Agility
Launch resources in minutes, not weeks.
5. Stop Spending on Data Centers
Focus on your business, not infrastructure.
6. Go Global in Minutes
Deploy worldwide with a few clicks using AWS Regions.

Key Cloud Benefits

BenefitWhat It Means
High AvailabilitySystem stays running even if parts fail (multiple AZs)
ElasticityAutomatically scale resources up/down based on demand

Fault ToleranceSystem continues operating even when components fail

Global ReachDeploy close to users worldwide for low latency

🔐 Domain 2: Security & Compliance (30%)

The Shared Responsibility Model (MOST TESTED TOPIC)

Think of it as: AWS = Security OF the Cloud | You = Security IN the Cloud

🟠 AWS Responsibility (OF the Cloud)

  • Physical data centers & security
  • Hardware & global infrastructure
  • Networking infrastructure
  • Hypervisor / virtualization layer
  • Managed service patching (e.g., RDS engine)
  • Edge locations & Regions

🔵 Customer Responsibility (IN the Cloud)

  • Data encryption & protection
  • IAM — users, roles, permissions
  • OS patching (on EC2)
  • Firewall / Security Group rules
  • Application code & updates
  • Network configuration (VPC, NACLs)

How Responsibility Shifts by Service Type

ServiceTypeCustomer DoesAWS Does
EC2IaaSOS patches, firewall, apps, dataHardware, hypervisor
RDSPaaSData, access controlOS patches, DB engine patches, backups
LambdaFaaSCode, data, IAMEverything else (OS, runtime, scaling)
S3ManagedData, bucket policies, encryption settingsInfrastructure, durability, availability
The exam will try to trick you. "Who patches the OS on RDS?" → AWS. "Who patches the OS on EC2?" → Customer.

⚡ Domain 3: Cloud Technology & Services (34%)

Compute Services

🖥️ Amazon EC2
Virtual servers in the cloud. YOU manage the OS. Choose instance type, size, OS. Most flexible compute option (IaaS).
⚡ AWS Lambda
Serverless. Run code without managing servers. Pay per request + compute time. Max 15 min per execution. Event-driven.
🐳 Amazon ECS
Run Docker containers on AWS. You manage the cluster (or use Fargate).
🐳 Amazon EKS
Managed Kubernetes. For teams already using Kubernetes.
🚀 AWS Fargate
Serverless containers. Run ECS/EKS without managing servers. Pay for resources used.
🌿 AWS Elastic Beanstalk
PaaS — upload your code, AWS handles deployment, scaling, load balancing. Still uses EC2 under the hood.
💡 AWS Lightsail
Simple virtual servers. Fixed monthly pricing. Great for small apps, blogs, dev environments. "Easy EC2."
📦 AWS Batch
Run batch computing jobs at scale. AWS manages the compute resources.
🖥️ AWS Outposts
Run AWS infrastructure ON-PREMISES. AWS hardware in YOUR data center. For hybrid cloud.

Auto Scaling

Automatically adds/removes EC2 instances based on demand. Ensures you have the right number of instances. Works with CloudWatch alarms. Provides both elasticity (scale out/in) and high availability.

Storage Services

🪣 Amazon S3 (Simple Storage Service)
Object storage. Unlimited storage. 99.999999999% (11 nines) durability. Store files, images, backups, static websites. Max object size: 5TB.
💾 Amazon EBS (Elastic Block Store)
Block storage for EC2. Like a virtual hard drive. Persists independently from EC2. One AZ only. Snapshots for backup.
📁 Amazon EFS (Elastic File System)
Shared file storage (NFS). Multiple EC2 instances can access simultaneously. Auto-scales. Linux only.
📁 Amazon FSx
Managed file systems. FSx for Windows (SMB). FSx for Lustre (high-performance computing).
❄️ S3 Glacier
Archive storage. Very cheap. Retrieval takes minutes to hours. For data you rarely access.
🔌 AWS Storage Gateway
Hybrid storage. Connects on-premises storage to AWS cloud storage (S3, Glacier).

Networking Services

🌐 Amazon VPC (Virtual Private Cloud)
Your own isolated network in AWS. You define subnets, route tables, gateways. Foundation of AWS networking.
🌍 Amazon CloudFront
CDN (Content Delivery Network). Caches content at edge locations worldwide. Reduces latency. Works with S3, EC2, ALB.
🔗 Amazon Route 53
DNS service. Translates domain names to IP addresses. Health checks & routing policies. Domain registration.
🔌 AWS Direct Connect
Dedicated private network connection from your data center to AWS. NOT over the internet. Low latency, consistent.
🔒 VPN (Site-to-Site)
Encrypted connection over the internet between your network and AWS VPC. Cheaper than Direct Connect.
🌐 AWS Global Accelerator
Routes traffic through AWS global network for better performance. Uses static IP addresses.

VPC Key Components

ComponentWhat It Does
SubnetsDivide VPC into sections. Public subnet (internet access) vs Private subnet (no direct internet).
Internet Gateway (IGW)Connects VPC to the internet. Required for public subnets.
NAT GatewayAllows private subnet instances to access internet (outbound only). Instances stay private.
Security GroupsInstance-level firewall. STATEFUL. Allow rules only (no deny). Default: deny all inbound.
NACLsSubnet-level firewall. STATELESS. Allow AND deny rules. Default: allow all.
VPC PeeringConnect two VPCs privately. No transitive peering (A↔B, B↔C ≠ A↔C).
Transit GatewayHub to connect multiple VPCs and on-premises networks. Solves peering complexity.
VPC EndpointsPrivate connection to AWS services without going through the internet.
VPC Flow LogsCapture IP traffic information in your VPC. For troubleshooting & security.
Security Groups = STATEFUL (return traffic auto-allowed). NACLs = STATELESS (must explicitly allow return traffic). This is a classic exam question.

💰 Domain 4: Billing, Pricing & Support (12%)

AWS Pricing Models

💳 Pay-As-You-Go
Pay only for what you use. No upfront costs. Like a utility bill. Most AWS services use this model.
📅 Reserved / Savings Plans
Commit to 1 or 3 years for significant discounts (up to 72%). Best for predictable workloads.
📦 Pay Less by Using More
Volume discounts. The more you use, the less per unit you pay (e.g., S3 storage tiers).

What You Pay For

You Pay ForYou DON'T Pay For
Compute time (EC2 hours, Lambda invocations)Inbound data transfer (data IN to AWS)
Storage (S3 GB, EBS GB)Data transfer between services in same AZ
Outbound data transfer (data OUT of AWS)Using the AWS Management Console
Provisioned resources (even if idle)Many services have a free tier
Inbound data = FREE. Outbound data = COSTS MONEY. This is a common exam question.

🎯 Final Exam Tips & Tricks

✅ Answer Every Question
No penalty for wrong answers. Never leave a question blank. Eliminate 2 options, guess between remaining 2.
🔑 Key Word Triggers
"Decouple" → SQS/SNS
"Serverless" → Lambda/Fargate/DynamoDB
"Audit/Who did what" → CloudTrail
"Monitor performance" → CloudWatch
"DDoS" → Shield
"Encryption keys" → KMS
"Compliance reports" → Artifact
"Cost recommendations" → Trusted Advisor
"Lift and shift" → MGN/Rehost
🧠 Common Traps
• CloudTrail ≠ CloudWatch (audit vs monitoring)
• Security Groups ≠ NACLs (stateful vs stateless)
• Shield Standard = free, Advanced = paid
• IAM is global, most services are regional
• S3 is global namespace, but data stays in Region
• Inbound data = free, Outbound = costs money
📝 Most Tested Topics
1. Shared Responsibility Model
2. IAM (users, roles, policies, MFA)
3. EC2 pricing models
4. S3 storage classes
5. Well-Architected Framework pillars
6. Support plans (who gets TAM?)
7. Security services (Shield, WAF, GuardDuty)
8. Billing tools (Budgets vs Cost Explorer)
🎯 Elimination Strategy
• If an answer mentions a service you've never heard of, it's probably wrong
• "Always" and "never" answers are usually wrong
• Look for the MOST correct answer, not just a correct one
• AWS always prefers managed/serverless solutions
⏰ Time Management
90 minutes / 65 questions = ~83 seconds per question.
Flag difficult questions and come back.
Don't spend more than 2 minutes on any question.
